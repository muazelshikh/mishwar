# 🚙 البوابة 4: بوابة أصحاب السيارات (Car Owners Portal) 🆕

**الرابط:** `mishwar.com/owner`

## الجمهور المستهدف

### النوع 1: مالك سيارة واحدة (لا يقود)
- لديه سيارة لكن لا يستخدمها
- يريد استثمارها

### النوع 2: مالك سيارات متعددة (شركة نقل صغيرة)
- لديه 2-20 سيارة
- يريد ضمها للمنصة

### النوع 3: مالك أسطول (شركة كبيرة)
- لديه 20+ سيارة
- يريد إدارة احترافية

## هيكل الصفحات

### الصفحات العامة
- `/owner` - صفحة الترحيب
- `/owner/register` - تسجيل
- `/owner/login` - دخول

### عملية التسجيل (Onboarding)

**`/owner/onboarding/step-1`** - معلومات المالك
- الاسم الكامل أو اسم الشركة
- نوع المالك (فرد/شركة)
- رقم الهوية أو السجل التجاري
- رقم الجوال
- البريد الإلكتروني
- العنوان

**`/owner/onboarding/step-2`** - معلومات الشركة (للشركات)
- اسم الشركة
- السجل التجاري
- الرخصة التجارية
- المدير المسؤول

**`/owner/onboarding/step-3`** - إضافة أول سيارة
- نوع وموديل السيارة
- السنة
- اللوحة
- الفئة (اقتصادي، VIP، إلخ)

**`/owner/onboarding/step-4`** - اختيار نموذج التشغيل
المالك يختار واحد أو أكثر:

**نموذج 1: تأجير يومي للسائقين 💰**
- السائق يدفع إيجار يومي ثابت
- المالك يستلم دخل ثابت
- السائق يحصل على كل إيرادات الرحلات

**نموذج 2: شراكة بالنسبة 🤝**
- بدون إيجار يومي
- تقسيم الإيرادات (مثلاً: 60% للسائق، 30% للمالك، 10% للمنصة)
- الكل يربح حسب الأداء

**نموذج 3: تخصيص للتأجير بدون سائق 🔑**
- السيارة للتأجير اليومي/الأسبوعي/الشهري
- المالك يحصل على نسبة من إيرادات التأجير

**نموذج 4: مختلط 🎯**
- السيارة تستخدم لكل النماذج
- النظام يحدد الاستخدام الأمثل

**`/owner/onboarding/step-5`** - معلومات البنك
- حساب لاستلام الإيرادات

**`/owner/onboarding/step-6`** - شروط وأحكام
- توقيع العقد الإلكتروني

### بعد التسجيل

#### `/owner/dashboard` - اللوحة الرئيسية
- إجمالي الإيرادات
- عدد السيارات النشطة
- أداء كل سيارة
- التنبيهات (صيانة، تأمين، إلخ)
- آخر النشاطات

#### `/owner/vehicles` - إدارة السيارات
- قائمة كل السيارات
- إضافة سيارة جديدة
- تعديل سيارة
- إيقاف سيارة مؤقتاً
- نقل بين النماذج

#### `/owner/vehicles/[id]` - تفاصيل سيارة

- تاريخ الإيرادات
- جدول الصيانة
- التقييمات
- الإحصائيات

#### `/owner/drivers` - السائقون على سياراتي
- قائمة السائقين الحاليين

- تقييمات
- استبدال سائق

#### `/owner/earnings` - الإيرادات

- مقارنات
- تنبؤات

#### `/owner/payments` - المدفوعات

#### `/owner/maintenance` - الصيانة

# 🎯 العلاقات بين البوابات

## كيف تتدفق البيانات؟

### مثال 1: راكب يطلب رحلة
```
الراكب (App) → طلب رحلة
   ↓
المنصة (Admin) → مطابقة سائق
   ↓
السائق (Driver) → استلام الطلب
   ↓
المالك (Owner) → استلام نسبته
   ↓
المنصة (Admin) → تسجيل الإيرادات
```

### مثال 2: شركة تشترك في خدمة
```
الشركة (Business) → عقد جديد
   ↓
الإدارة (Admin) → اعتماد العقد
   ↓
السائقون (Driver) → استلام الجدول
   ↓
الموظفون (App) → استخدام الخدمة
```

### مثال 3: مالك يضم سيارة
```
المالك (Owner) → إضافة سيارة
   ↓
الإدارة (Admin) → اعتماد السيارة
   ↓
السائقون (Driver) → اختيار السيارة
   ↓
الركاب (App) → استخدام الخدمة
   ↓
الإيرادات تتوزع بين المالك والسائق والمنصة
```
